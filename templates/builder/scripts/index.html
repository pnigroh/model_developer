Class Migrator Scripts

Two-step pipeline: generate Django models from a CSV export, then deploy into a project

Step 1 Generate Models from CSV

Upload a semicolon-delimited CSV export of your class/field definitions. The script converts every class into a Django model and outputs a complete app folder (models.py, admin.py, apps.py, __init__.py).

Parameters
-i / --inputCSV file required
-o / --outputOutput directory path required
-a / --app-nameDjango app name optional
Run Generate Script
Step 2 Deploy App to Django Project

Takes the generated app folder (or a zip) and wires it into an existing Django/Django-CMS project: patches settings.py and urls.py, generates views and templates, and runs migrations.

Parameters
-i / --inputApp folder or zip required
-t / --targetProject root path required
-a / --app-nameOverride app name optional
-m / --modeupdate / replace / new-only optional
--dry-runPreview only, no writes optional
--no-migrateSkip migrations optional
Run Deploy Script
